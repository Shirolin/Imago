# Crop 工作区设计评审与优化建议

基于目前的界面截图，Imago 的裁剪工作区已经具备了良好的基础结构（清晰的左侧状态、中间预览、右侧控制流）。为了提升“高级感”和工具的“专业性”，建议从以下几个维度进行深度优化：

## 1. 视觉一致性与节奏感 (Visual Consistency)

目前的右侧属性面板在元素排布上略显零散：

- **变换按钮归一化**: “旋转”、“水平翻转”是图标按钮，而“垂直翻转”和“重置”是文字按钮。建议全部统一为**带标签的图标按钮**或**纯图标群组**。
- **分界线与留白**: 各个 Section（纵横比、变换、导出）之间的间距可以更灵活一些，使用更细微的分割方式（如背景色块微弱差异而非实线）。
- **色彩语义**: 目前“自由”按钮使用了高亮青色，而主动作使用橙色。建议建立一套核心主色，减少多种高饱和度颜色的冲突。

## 2. 交互体验增强 (UX Enhancements)

- **智能比例预览**: 在点击 1:1 或 16:9 时，可以弹出一个微缩的比例示意图，或者让按钮本身具有比例感（比如按钮的形状对应其比例）。
- **坐标与尺寸可视化**: 裁剪框右下角的 `453 x 602` 很实用，但如果在侧边栏也实时显示 `X, Y, W, H` 的数值输入框，将极大地满足精确裁剪的需求。
- **画布控制**: 增加“自适应窗口”和“1:1 像素显示”的缩放控制，对于大图裁剪非常关键。
- **快捷键指引**: 在按钮悬停或侧边栏底部显示快捷键（如 `R` 旋转，`V` 垂直翻转，`Enter` 应用）。

## 3. 功能补全 (Feature Completeness)

- **九宫格辅助线**: 目前裁剪框内已有辅助线，可以提供更多模式（如黄金分割线、对角线）。
- **导出质量预估**: 在调整“画质”滑块时，如果能实时估算预计文件大小（基于上一轮处理结果），体验会非常“惊艳”。
- **批量同步提示**: 如果当前选择了多张图片，应用裁剪时应更明显地提示“此裁剪区域将应用至所有选中项”。

## 优先推荐的下一步行动

1. **[UI] 侧边栏布局重构**: 统一变换工具组的视觉样式。
2. **[UI] 增强比例选择器**: 为不同比例增加视觉草图，让选择更直观。
3. **[UX] 悬浮动作条优化**: 将“撤销/重做”移至靠近画布的浮动位置，减少鼠标长距离移动。
