# Imago 项目 UI/UX 优化执行计划

本计划旨在系统性地利用 Imago 项目中已配置的专属技能（Skills），对现有的组件、页面布局和样式进行从宏观到微观的全面升级。

## 第一阶段：全局评估与规范化 (Foundation)
**目标：** 统一视觉语言，找出当前体验的痛点。

1. **UX 审查与反馈**
   * **使用技能：** `critique`
   * **执行对象：** `src/views/HomeView.vue`、`src/components/layout/WorkspaceLayout.vue`
   * **操作建议：** 激活该技能，让其评估当前主工作区的信息架构和视觉层级，找出不合理的设计并记录改进点。
2. **设计系统标准化**
   * **使用技能：** `normalize`
   * **执行对象：** `tailwind.config.js`、`src/style.css` 以及 `src/components/common/` 下的基础组件（如 `AppButton.vue`, `AppModal.vue`）。
   * **操作建议：** 让技能审查基础组件，确保颜色、圆角、阴影、字体层级等设计令牌（Design Tokens）的一致性，消除“东拼西凑”的视觉感。

## 第二阶段：核心功能视图重塑 (Core Redesign)
**目标：** 提升核心工具页面的专业感和“高级感”。

1. **界面视觉重构**
   * **使用技能：** `frontend-design`
   * **执行对象：** `src/views/CompressView.vue`、`src/views/FiltersView.vue` 等核心图像处理视图。
   * **操作建议：** 针对功能复杂的页面，要求技能重新设计面板布局，摆脱传统的“表单堆砌”感，创造具有现代工具软件质感（如类 macOS 或精美 Web App）的界面。
2. **文案与微文案清晰化**
   * **使用技能：** `clarify`
   * **执行对象：** 各个工具视图中的提示说明（如 `AppTip.vue`）、表单标签、空状态提示。
   * **操作建议：** 优化图像处理参数的说明，使专业术语更容易被普通用户理解。

## 第三阶段：细节打磨与交互增强 (Polish & Delight)
**目标：** 让界面从“可用”变为“精致”和“令人愉悦”。

1. **像素级对齐与细节修复**
   * **使用技能：** `polish`
   * **执行对象：** `src/components/common/ImageCard.vue`、`ImageUpload.vue`。
   * **操作建议：** 修复组件内部的间距（Padding/Margin）不均、对齐偏差、边框粗细不一等问题。
2. **添加过渡与微交互**
   * **使用技能：** `animate`
   * **执行对象：** 文件上传拖拽状态、图片处理进度条、参数滑动条（`AppSlider.vue`）的交互反馈。
   * **操作建议：** 为状态切换（如“处理中”到“处理完成”）添加优雅的缓动动画，提升软件的响应感。
3. **注入个性与温度**
   * **使用技能：** `delight`
   * **执行对象：** 处理成功后的反馈状态、空工作区状态（Empty States）。
   * **操作建议：** 在无图可处理或处理完美完成时，增加一些有趣的视觉元素或微交互，提升用户情绪。

## 第四阶段：边缘场景与性能优化 (Robustness)
**目标：** 确保工具在极端情况下依然表现完美。

1. **响应式与多端适配**
   * **使用技能：** `adapt`
   * **执行对象：** `WorkspaceLayout.vue` 及各工具栏组件。
   * **操作建议：** 确保侧边栏和图片预览区在小屏幕（如平板设备）下能自适应折叠或重新排列，不出现内容溢出。
2. **极端状态与错误处理**
   * **使用技能：** `harden`
   * **执行对象：** 文件上传校验逻辑、超大图片渲染失败时的占位图。
   * **操作建议：** 完善格式不支持、图片过大等情况下的 UI 表现，确保错误提示优雅且不破坏布局。
3. **渲染性能优化**
   * **使用技能：** `optimize`
   * **执行对象：** `src/views/GridView.vue`、`src/components/common/ImageCompare.vue`。
   * **操作建议：** 针对多图网格和实时滤镜对比功能，优化 DOM 结构和 CSS 渲染属性（如使用 `will-change` 或 `transform` 代替导致重排的属性），确保操作流畅。
